libtdk

Terminal Development Kit

* This documentation refers to v8.0.0.
View Source Code

About

A simple C99 terminal manipulation library for Linux and Windows that is capable of handling some terminal properties, styles and key readings with UTF-8 encoding.

Designed to be an alternative to curses-like libraries, it offers a solid base for creating general purpose terminal based softwares, while leaving room to full-feature libraries to be built upon for more specific use cases.

Standard Streams

A process running in a terminal is always connected to three standard data streams, those are:

  • standard input (tdk_Stream_Input) from where it may read events or characters.
  • standard output (tdk_Stream_Output): to where it may send regular tense messages.
  • standard error (tdk_Stream_Error): to where it may send error tense messages.

By default, the standard streams are usually connected to the process (TTY). However, all operating systems allow you to redirect them to other files by using shell scripting languages. You can check the status of those streams by using the function tdk_isTTY.

Functions

tdk_isTTY Function

Declaration

int tdk_isTTY(int stream);

Brief

Checks if a standard stream is connected to a terminal process (TTY).

Parameters

  • stream: the standard stream to be checked. It may be tdk_Stream_Input, tdk_Stream_Output or tdk_Stream_Error.

Example

#include <tdk.h>

#define BOOLEAN(a_value) (a_value ? "true" : "false")

int main(void) {
  tdk_writeLine("TTY Statuses");
  tdk_writeLine("------------");
  tdk_writeLine(":: Input : %s.\n", BOOLEAN(tdk_isTTY(tdk_Stream_Input)));
  tdk_writeLine(":: Output: %s.\n", BOOLEAN(tdk_isTTY(tdk_Stream_Output)));
  tdk_writeLine(":: Error : %s.\n", BOOLEAN(tdk_isTTY(tdk_Stream_Error)));
  return 0;
}
