libtdk

About

A simple C99 terminal manipulation library for Linux and Windows that is capable of handling some terminal properties, styles and key readings with UTF-8 encoding.

Designed to be an alternative to curses-like libraries, it offers a solid base for creating general purpose terminal based softwares, while leaving room to full-feature libraries to be built upon for more specific use cases.

Introduction

Terminals are a basic human-computer interface that allows people to interact with an operating system through the use of a shell.

Its properties can be manipulated through the use of a specific language of codes, called "virtual terminal sequences" or "ANSI escape sequences", and exclusive API calls to the operating system they are running into.

The terminal also manipulates its own data streams. However, with the advent and evolution of Unicode, the operating systems have also diverged in the way they encode graphemes, creating a serie of issues related to character encoding.

Inspired by the simple interface present in the C# language, this library was developed to provide the missing features from the standard C library to manipulate terminals in a way that is cross-platform and that uses UTF-8 encoding.

Standard Streams

Every terminal process contains three standard data streams from which it can receive and send information:

  • standard input (tdk_Stream_Input): it is used to obtain events and input from a user.
  • standard output (tdk_Stream_Output): it is used to output regular tense messages.
  • standard error (tdk_Stream_Error): it is used to output error tense messages.

These streams may be connected to a terminal window, therefore creating a TTY, or to a file. The flow of those streams can be changed by the use of shell scripting language. You may check the status of each stream by using tdk_isTTY function.
