About

A simple C99 terminal manipulation library for Linux and Windows that can be used to develop cross-platform terminal application using C and C++. In summary, it includes features to manipulate terminal properties, styles and key events encoded in UTF-8.

Designed to be an alternative to curses-like libraries, it offers a solid base for creating general purpose terminal based softwares, while leaving room to full-feature libraries to be built upon for more specific use cases.

Throughout this documentation, you will learn the basics of the terminal and how to use libtdk's functions to manipulate it.

Reserved Names

This library reserves all names that starts with tdk possibly prefixed by an underline to indicate that it is isolated to its own implementation.

What Is A Terminal

The terminal is a simple human-computer interface that allow people to interact with an operating system through a text based-command interpreter software called "shell".

In the old days, a terminal used to be a real machine, but in the modern era its behavior is emulated through a software called "terminal emulator". The term "terminal" in this documentation will assume this meaning.

Terminal Components

Each operating system decides the components that builds their own terminal API. However, there are some components that are common to all software of this type. Those are:

Window

A graphical window on a desktop environment or the whole monitor in case of embeded operating system terminals. It displays the output of an active running process. The terminal also keeps a buffer with the contents and properties to display.

The window is composed by multiple cell of characters, spread horizontally, in columns, and vertically, in rows. To get the coordinate of a cell, an origin position at the top left corner of the window, where column and row are both zero, is considered. From there, the values of column and row increase going right and down respectively.

The window may appear in many different dimensions. You can get the total of columns and rows that compose the terminal window at a given moment by using the tdk_getWindowDimensions function.

Cursor

A special coordinate in the window that is used as reference when writting output onto the window. The terminal usually highlights this coordinate in the window using a specific shape that may be visible or not. You can manipulate those cursor properties by using the tdk_getCursorCoordinate, tdk_setCursorShape and tdk_setCursorVisibility functions.

Standard Streams

Any terminal process that is active at a given moment has access to three standard data streams, those are:

  • standard input or stdin (tdk_Stream_Input): from where it can read characters/events.
  • standard output or stdout (tdk_Stream_Output): where it can output regular tense messages.
  • standard error or stderr (tdk_Stream_Error): where it can output error tense messages.

Each of these streams may be connected to a terminal (TTY) that will be providing user interactions or be a file in case of stream redirections and pipelines. You can check if a stream is a TTY by using the tdk_isTTY function.

The standard output and error streams are writtable and you can write to them by using the tdk_write, tdk_writeLine, tdk_writeError and tdk_writeErrorLine.

Input Devices

Even though not exactly part of a terminal nowadays, input devices like a keyboard and mouse send character/events to a terminal input buffer and allow an user to interact with it.

For simplicity and compatibility, this library only allows you to track synchronous key events by using the tdk_readKeyEvent.

Colors And Layers

Modern terminals usually support two palettes:

  • xterm palette: a palette of 256 specific colors. The first 16 colors have names and contain both a regular and light variants, these are also the ones defined in a terminal theme, thus they provide the best compatibility with those themes:
    • black (tdk_Color_Black/tdk_Color_LightBlack): same as values 0 and 8.
    • red (tdk_Color_Red/tdk_Color_LightRed): same as values 1 and 9.
    • green (tdk_Color_Green/tdk_Color_LightGreen): same as values 2 and 10.
    • yellow (tdk_Color_Yellow/tdk_Color_LightYellow): same as values 3 and 11.
    • blue (tdk_Color_Blue/tdk_Color_LightBlue): same as values 4 and 12.
    • magenta (tdk_Color_Magenta/tdk_Color_LightMagenta): same as values 5 and 13.
    • cyan (tdk_Color_Cyan/tdk_Color_LightCyan): same as values 6 and 14.
    • white (tdk_Color_White/tdk_Color_LightWhite): same as values 7 and 15.

    You can set a color from this palette by using the tdk_set256Color function.

  • hex/rgb palette: this palette contains over 16.500.000 colors and it is very popular in the graphic design field. Terminals allow you to use them through a "truecolor" feature. You can set a color from this palette by using the tdk_setHEXColor and tdk_setRGBColor functions.

Any color that you choose to apply must affect a certain terminal layer, that can be: the foreground (tdk_Layer_Foreground) or the background (tdk_Layer_Background).

Effects

Terminals can support a range of text effects. By using the tdk_setEffect function, you can set the following effects:

  • Italic (tdk_Effect_Italic): it makes the text look curly. It may require the use of a font with italic style to be seen.
  • Underline (tdk_Effect_Underline): it draws a horizontal line below the text.
  • Blinking (tdk_Effect_Blinking): it makes the text blink indefinetely over time.
  • Reverse Video(tdk_Effect_ReverseVideo): it swaps the foreground and background colors.
  • Conceal(tdk_Effect_Conceal): it makes the text invisible or hard to see.
  • Strikethrough(tdk_Effect_Strikethrough): it draws a horizontal line crossing into the middle of the text.
